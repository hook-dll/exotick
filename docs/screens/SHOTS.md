# Screenshots

The 6 PNGs in this folder power the README. They're regenerated by an
automated Playwright script — no manual capture needed.

## Regenerate

```bash
# One-time setup
npx playwright install chromium

# Build server + client (needed by the capture script)
cd server && npx tsc && cd ..
cd client && npx vite build && cd ..

# Capture
node docs/screens/capture.mjs
```

The script spins up an isolated exotick instance against a throwaway temp
data folder, seeds sample data, drives Playwright through six states, and
writes PNGs into this folder. Your real DB is never touched.

## The six shots

| File | State |
|---|---|
| `01-edit-mode.png` | Edit Mode with the three sample sections (Auth blue, Checkout green, Admin purple), a case with 📄 selected so its markdown description previews on the right. |
| `02-compose.png` | Compose Test Run — sticky top panel with Run/Runner filled in and "11 / 14 selected"; Auth + Checkout section-headers checked. |
| `03-active-run.png` | Active Test Run mid-flight — 3 Pass + 1 Fail marked, progress bar filled, an unmarked case previewed on the right. |
| `04-history-detail.png` | Finished run — Total/Pass/Fail/Skip summary + colored per-section groups with status badges. |
| `05-settings.png` | Settings — Editor password empty state. |
| `06-pastel.png` | Pastel easter egg — soft pastel wash on the dashboard with "☀️ Back to light" + "🎨 New color" in the sidebar. |

## Adjusting the shots

Everything is scripted in `capture.mjs`. Change the case that gets previewed,
which sections get checked, or the viewport size, all in one file. Re-run to
regenerate.

## Manual capture fallback

If Playwright isn't an option (offline install, corporate policy), any
screenshot tool works — same target states, same filenames. Browser at
100% zoom, viewport ~1440×900, light theme, load sample data first.
